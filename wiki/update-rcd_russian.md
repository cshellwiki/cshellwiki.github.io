# [Linux] Bash update-rc.d: [управление скриптами инициализации]

## Обзор
Команда `update-rc.d` используется для управления скриптами инициализации в системах на основе Debian. Она позволяет добавлять, удалять или изменять порядок запуска скриптов при загрузке системы.

## Использование
Основной синтаксис команды выглядит следующим образом:

```bash
update-rc.d [options] [arguments]
```

## Общие параметры
- `defaults` — добавляет скрипт в стандартные уровни запуска.
- `remove` — удаляет скрипт из уровней запуска.
- `enable` — включает скрипт для автоматического запуска.
- `disable` — отключает автоматический запуск скрипта.
- `force` — принудительно выполняет действие, даже если это может вызвать проблемы.

## Общие примеры
1. Добавление скрипта в стандартные уровни запуска:
   ```bash
   sudo update-rc.d myscript defaults
   ```

2. Удаление скрипта из уровней запуска:
   ```bash
   sudo update-rc.d myscript remove
   ```

3. Включение скрипта для автоматического запуска:
   ```bash
   sudo update-rc.d myscript enable
   ```

4. Отключение автоматического запуска скрипта:
   ```bash
   sudo update-rc.d myscript disable
   ```

5. Принудительное удаление скрипта:
   ```bash
   sudo update-rc.d myscript remove --force
   ```

## Советы
- Перед внесением изменений всегда делайте резервные копии ваших скриптов.
- Проверяйте порядок запуска скриптов, чтобы избежать конфликтов между ними.
- Используйте `update-rc.d` с осторожностью, особенно с параметром `--force`, чтобы не нарушить работу системы.
