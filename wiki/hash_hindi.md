# [लिनक्स] Bash hash उपयोग: कमांड के लिए कैश प्रबंधन

## Overview
`hash` कमांड का उपयोग शेल में कमांड के नामों और उनके पथ को कैश करने के लिए किया जाता है। यह कमांड यह सुनिश्चित करता है कि जब आप किसी कमांड को चलाते हैं, तो शेल उसे तेजी से खोज सके।

## Usage
`hash` कमांड का मूल सिंटैक्स इस प्रकार है:

```bash
hash [options] [arguments]
```

## Common Options
- `-r`: कैश को रीसेट करता है, अर्थात् सभी कैश की गई प्रविष्टियों को हटा देता है।
- `-p`: एक विशेष कमांड के लिए पथ को सेट करता है।
- `-l`: कैश की गई सभी कमांडों की सूची दिखाता है।

## Common Examples
1. **कैश को रीसेट करना:**
   ```bash
   hash -r
   ```

2. **विशेष कमांड के लिए पथ सेट करना:**
   ```bash
   hash -p /usr/local/bin/mycommand mycommand
   ```

3. **कैश की गई कमांडों की सूची देखना:**
   ```bash
   hash -l
   ```

4. **विशिष्ट कमांड का पथ देखना:**
   ```bash
   hash mycommand
   ```

## Tips
- जब आप नए कमांड इंस्टॉल करते हैं, तो `hash -r` का उपयोग करना सुनिश्चित करें ताकि शेल को नवीनतम पथ जानकारी मिल सके।
- यदि आप किसी कमांड के लिए विशेष पथ सेट करते हैं, तो यह सुनिश्चित करें कि वह पथ सही और कार्यशील हो।
- नियमित रूप से कैश की गई कमांडों की सूची की जांच करें ताकि आप यह सुनिश्चित कर सकें कि आपके पास सही और अद्यतन जानकारी है।
