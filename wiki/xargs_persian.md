# [لینوکس] Bash xargs استفاده: اجرای دستورات بر روی ورودی‌های استاندارد

## Overview
دستور `xargs` در Bash برای تبدیل ورودی‌های استاندارد به آرگومان‌های خط فرمان استفاده می‌شود. این ابزار به شما این امکان را می‌دهد که خروجی یک دستور را به عنوان ورودی برای دستور دیگری استفاده کنید، که می‌تواند در پردازش دسته‌ای داده‌ها بسیار مفید باشد.

## Usage
سینتکس پایه دستور `xargs` به صورت زیر است:

```bash
xargs [options] [arguments]
```

## Common Options
- `-n N`: تعداد آرگومان‌هایی که به هر بار اجرای دستور ارسال می‌شود را مشخص می‌کند.
- `-d DELIMITER`: مشخص می‌کند که ورودی‌ها با چه کاراکتری جدا شوند.
- `-p`: قبل از اجرای هر دستور، از کاربر تأیید می‌گیرد.
- `-0`: ورودی‌ها را به صورت null-terminated می‌خواند، که برای پردازش فایل‌هایی با نام‌های خاص مفید است.

## Common Examples
### مثال 1: حذف فایل‌ها
برای حذف فایل‌هایی که با پسوند `.tmp` تمام می‌شوند، می‌توانید از دستور زیر استفاده کنید:

```bash
find . -name "*.tmp" | xargs rm
```

### مثال 2: شمارش تعداد خطوط در چند فایل
برای شمارش تعداد خطوط در فایل‌های متنی، می‌توانید از این دستور استفاده کنید:

```bash
ls *.txt | xargs wc -l
```

### مثال 3: اجرای یک دستور با آرگومان‌های چندگانه
اگر بخواهید یک دستور را با چندین آرگومان اجرا کنید، می‌توانید از گزینه `-n` استفاده کنید:

```bash
echo "file1 file2 file3" | xargs -n 1 cp -v /source/directory/
```

## Tips
- همیشه ورودی‌ها را بررسی کنید تا از بروز خطاهای احتمالی جلوگیری کنید.
- از گزینه `-p` برای تأیید قبل از اجرای دستورات خطرناک استفاده کنید.
- برای پردازش فایل‌هایی با نام‌های خاص، از گزینه `-0` استفاده کنید تا از مشکلات ناشی از فضاهای خالی یا کاراکترهای خاص جلوگیری شود.
