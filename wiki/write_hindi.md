# [Linux] Bash write उपयोग: उपयोगकर्ताओं के बीच संदेश भेजना

## Overview
`write` कमांड का उपयोग एक उपयोगकर्ता से दूसरे उपयोगकर्ता को सीधे संदेश भेजने के लिए किया जाता है। यह कमांड विशेष रूप से तब उपयोगी होती है जब आप किसी अन्य उपयोगकर्ता को तुरंत जानकारी देना चाहते हैं, जबकि वे सिस्टम पर लॉग इन हैं।

## Usage
`write` कमांड का मूल सिंटैक्स इस प्रकार है:

```bash
write [options] [username] [tty]
```

यहाँ, `username` वह उपयोगकर्ता है जिसे आप संदेश भेजना चाहते हैं और `tty` वह टर्मिनल है जिसमें वह उपयोगकर्ता लॉग इन है। यदि `tty` निर्दिष्ट नहीं किया गया है, तो `write` स्वचालित रूप से उपयोगकर्ता के वर्तमान टर्मिनल का पता लगाएगा।

## Common Options
- `-n`: संदेश भेजने से पहले उपयोगकर्ता के टर्मिनल को चेक नहीं करता है।
- `-s`: संदेश को केवल एक बार भेजता है, बिना किसी अन्य जानकारी के।
- `-f`: संदेश को फ़ाइल से पढ़ता है।

## Common Examples

1. किसी उपयोगकर्ता को संदेश भेजना:
   ```bash
   write john
   Hello John, how are you?
   ```

2. एक विशेष टर्मिनल पर संदेश भेजना:
   ```bash
   write john pts/1
   Don't forget the meeting at 3 PM.
   ```

3. संदेश को फ़ाइल से भेजना:
   ```bash
   write john < message.txt
   ```

4. संदेश भेजने से पहले उपयोगकर्ता के टर्मिनल की जांच किए बिना:
   ```bash
   write -n john
   Urgent: Please check your email.
   ```

## Tips
- सुनिश्चित करें कि प्राप्तकर्ता का टर्मिनल खुला है, अन्यथा संदेश नहीं पहुँच पाएगा।
- यदि आप एक लंबा संदेश भेज रहे हैं, तो उसे पहले एक फ़ाइल में लिखें और फिर उसे भेजें।
- ध्यान रखें कि `write` कमांड से भेजे गए संदेश सभी उपयोगकर्ताओं द्वारा देखे जा सकते हैं जो उस टर्मिनल पर हैं।
