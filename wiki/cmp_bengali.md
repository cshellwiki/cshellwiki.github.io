# [লিনাক্স] Bash cmp ব্যবহার: ফাইলের মধ্যে পার্থক্য নির্ধারণ

## Overview
`cmp` কমান্ডটি দুটি ফাইলের মধ্যে পার্থক্য নির্ধারণ করতে ব্যবহৃত হয়। এটি ফাইলের প্রথম ভিন্ন বাইটের অবস্থান এবং সেই ভিন্নতার কারণে ফাইল দুটি কতটা আলাদা তা দেখায়।

## Usage
`cmp` কমান্ডের মৌলিক সিনট্যাক্স হল:

```bash
cmp [options] [arguments]
```

## Common Options
- `-l`: ভিন্ন বাইটগুলোর অবস্থান এবং মান দেখায়।
- `-s`: কোনো আউটপুট ছাড়াই কেবলমাত্র ফাইল দুটি সমান কিনা তা পরীক্ষা করে।
- `-i <skip>`: প্রথম `<skip>` বাইট বাদ দিয়ে তুলনা শুরু করে।
- `-n <bytes>`: শুধুমাত্র প্রথম `<bytes>` বাইট তুলনা করে।

## Common Examples
নিচে কিছু সাধারণ উদাহরণ দেওয়া হল:

### উদাহরণ 1: দুটি ফাইল তুলনা করা
```bash
cmp file1.txt file2.txt
```
এই কমান্ডটি `file1.txt` এবং `file2.txt` এর মধ্যে পার্থক্য দেখাবে।

### উদাহরণ 2: ভিন্ন বাইটের অবস্থান এবং মান দেখানো
```bash
cmp -l file1.txt file2.txt
```
এই কমান্ডটি ভিন্ন বাইটগুলোর অবস্থান এবং তাদের মান দেখাবে।

### উদাহরণ 3: আউটপুট ছাড়া তুলনা করা
```bash
cmp -s file1.txt file2.txt
```
এই কমান্ডটি কেবলমাত্র জানাবে যে ফাইল দুটি সমান কিনা, কিন্তু কোনো আউটপুট দিবে না।

### উদাহরণ 4: প্রথম 10 বাইট তুলনা করা
```bash
cmp -n 10 file1.txt file2.txt
```
এই কমান্ডটি প্রথম 10 বাইট তুলনা করবে এবং পার্থক্য থাকলে দেখাবে।

## Tips
- `cmp` ব্যবহার করার সময় নিশ্চিত করুন যে আপনি সঠিক ফাইলের নাম এবং পাথ ব্যবহার করছেন।
- যদি আপনি আউটপুট দেখতে না চান তবে `-s` অপশনটি ব্যবহার করুন।
- বড় ফাইলের ক্ষেত্রে, `-n` অপশন ব্যবহার করে নির্দিষ্ট সংখ্যক বাইট তুলনা করা কার্যকর হতে পারে।
