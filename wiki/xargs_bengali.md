# [লিনাক্স] Bash xargs ব্যবহার: আউটপুটকে ইনপুটে রূপান্তর করা

## Overview
xargs কমান্ডটি একটি শক্তিশালী টুল যা স্ট্যান্ডার্ড ইনপুট থেকে আউটপুট গ্রহণ করে এবং সেটিকে আর্গুমেন্ট হিসেবে অন্য একটি কমান্ডে পাঠায়। এটি বিশেষভাবে উপকারী যখন আপনি একটি দীর্ঘ আর্গুমেন্ট লিস্ট তৈরি করতে চান যা অন্য কমান্ডের জন্য প্রয়োজন।

## Usage
xargs কমান্ডের মৌলিক সিনট্যাক্স হলো:

```bash
xargs [options] [arguments]
```

## Common Options
- `-n N`: প্রতি Nটি আর্গুমেন্টে একটি কমান্ড চালায়।
- `-d DELIMITER`: আর্গুমেন্টগুলিকে নির্দিষ্ট ডেলিমিটার দ্বারা বিভক্ত করে।
- `-I {}`: আর্গুমেন্টগুলিকে `{}` এর মাধ্যমে প্রতিস্থাপন করে।
- `-p`: প্রতিটি কমান্ড চালানোর আগে ব্যবহারকারীর অনুমতি চায়।
- `-0`: null ক্যারেক্টার দ্বারা পৃথক করা আর্গুমেন্ট গ্রহণ করে (যেমন `find` কমান্ডের সাথে ব্যবহার করার সময়)।

## Common Examples
### উদাহরণ ১: ফাইলের নাম মুছে ফেলা
```bash
find . -name "*.tmp" | xargs rm
```
এই কমান্ডটি বর্তমান ডিরেক্টরিতে সব `.tmp` ফাইল খুঁজে বের করে এবং সেগুলি মুছে ফেলে।

### উদাহরণ ২: ফাইলের কপি তৈরি করা
```bash
echo "file1.txt file2.txt" | xargs -n 1 cp -t /backup/
```
এই কমান্ডটি `file1.txt` এবং `file2.txt` ফাইলগুলির একটি কপি `/backup/` ডিরেক্টরিতে তৈরি করে।

### উদাহরণ ৩: ফাইলের সংখ্যা গণনা করা
```bash
ls | xargs -n 1 echo "File:"
```
এই কমান্ডটি বর্তমান ডিরেক্টরির প্রতিটি ফাইলের নামের আগে "File:" যুক্ত করে আউটপুট দেখায়।

## Tips
- `xargs` ব্যবহার করার সময়, নিশ্চিত করুন যে আপনার ইনপুট সঠিকভাবে ফরম্যাট করা হয়েছে।
- বড় আর্গুমেন্ট লিস্টের জন্য `-n` অপশন ব্যবহার করে আর্গুমেন্টগুলিকে ছোট গ্রুপে ভাগ করুন।
- `-0` অপশন ব্যবহার করুন যখন আপনি ফাইলের নামগুলিতে স্পেস বা বিশেষ ক্যারেক্টার আছে।
- `-I {}` ব্যবহার করে আর্গুমেন্টগুলিকে কাস্টম প্লেসহোল্ডার দিয়ে প্রতিস্থাপন করতে পারেন, যা আরও নমনীয়তা প্রদান করে।
