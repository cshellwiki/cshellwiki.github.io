# [Linux] Bash tmux の使い方: ターミナルマルチプレクサ

## 概要
tmuxは、ターミナルマルチプレクサであり、1つのターミナルウィンドウ内で複数のセッションを管理することができます。これにより、複数のシェルを同時に操作したり、セッションを切り離して後で再接続したりすることが可能です。

## 使用法
tmuxの基本的な構文は以下の通りです。

```bash
tmux [options] [arguments]
```

## 一般的なオプション
- `new`: 新しいtmuxセッションを作成します。
- `attach`: 既存のtmuxセッションに接続します。
- `detach`: 現在のtmuxセッションから切り離します。
- `list-sessions`: 現在のtmuxセッションのリストを表示します。

## 一般的な例
以下に、tmuxの一般的な使用例を示します。

### 新しいセッションを作成する
```bash
tmux new -s mysession
```

### 既存のセッションに接続する
```bash
tmux attach -t mysession
```

### セッションを切り離す
tmuxセッション内で、以下のキーボードショートカットを使用します。
```
Ctrl + b, d
```

### 現在のセッションをリスト表示する
```bash
tmux list-sessions
```

## ヒント
- tmuxを使用する際は、キーボードショートカットを覚えると効率的です。
- セッションを切り離すことで、長時間のプロセスを実行している間もターミナルを他の作業に使用できます。
- tmuxの設定ファイル（`~/.tmux.conf`）を作成して、カスタマイズを行うことができます。
