# [लिनक्स] Bash xargs उपयोग: कमांड लाइन पर डेटा को प्रोसेस करना

## Overview
xargs एक शक्तिशाली Bash कमांड है जो मानक इनपुट से डेटा को पढ़ता है और उसे एक या अधिक कमांड के रूप में पास करता है। यह विशेष रूप से तब उपयोगी होता है जब आप एक साथ कई आर्ग्युमेंट्स के साथ किसी कमांड को चलाना चाहते हैं।

## Usage
xargs का मूल सिंटैक्स इस प्रकार है:

```bash
xargs [options] [arguments]
```

## Common Options
- `-n N`: हर बार N आर्ग्युमेंट्स के साथ कमांड चलाता है।
- `-d DELIM`: इनपुट को एक विशेष डिलिमिटर के आधार पर विभाजित करता है।
- `-I REPLACE`: एक प्लेसहोल्डर के रूप में REPLACE का उपयोग करते हुए कमांड में आर्ग्युमेंट्स को प्रतिस्थापित करता है।
- `-p`: प्रत्येक कमांड चलाने से पहले उपयोगकर्ता से पुष्टि मांगता है।

## Common Examples
1. **फाइलों को डिलीट करना**:
   ```bash
   find . -name "*.tmp" | xargs rm
   ```
   यह कमांड वर्तमान डायरेक्टरी में सभी `.tmp` फाइलों को ढूंढता है और उन्हें डिलीट करता है।

2. **फाइलों की लिस्टिंग**:
   ```bash
   echo "file1.txt file2.txt file3.txt" | xargs ls -l
   ```
   यह कमांड दी गई फाइलों की विस्तृत जानकारी दिखाता है।

3. **फाइलों को कॉपी करना**:
   ```bash
   find . -name "*.jpg" | xargs -I {} cp {} /path/to/destination/
   ```
   यह कमांड सभी `.jpg` फाइलों को निर्दिष्ट गंतव्य में कॉपी करता है।

4. **एक्सटेंशन बदलना**:
   ```bash
   find . -name "*.txt" | xargs -n 1 bash -c 'mv "$0" "${0%.txt}.md"'
   ```
   यह कमांड सभी `.txt` फाइलों को `.md` में बदलता है।

## Tips
- xargs का उपयोग करते समय, सुनिश्चित करें कि आपके इनपुट में कोई विशेष कैरेक्टर न हो जो कमांड को बाधित कर सके।
- `-n` विकल्प का उपयोग करें ताकि आप एक बार में कितने आर्ग्युमेंट्स पास कर रहे हैं, उसे नियंत्रित कर सकें।
- जब भी संभव हो, `-I` विकल्प का उपयोग करें ताकि आप अधिक लचीले ढंग से कमांड को कस्टमाइज़ कर सकें।
