# [لینوکس] Bash udevadm استفاده: مدیریت دستگاه‌ها در سیستم

## Overview
دستور `udevadm` ابزاری است که برای مدیریت و نظارت بر دستگاه‌های متصل به سیستم در لینوکس استفاده می‌شود. این دستور به شما امکان می‌دهد تا اطلاعات مربوط به دستگاه‌ها را مشاهده کنید و تغییرات در سیستم را پیگیری کنید.

## Usage
سینتکس پایه دستور `udevadm` به شکل زیر است:

```bash
udevadm [options] [arguments]
```

## Common Options
- `info`: نمایش اطلاعات مربوط به یک دستگاه خاص.
- `trigger`: فعال‌سازی رویدادهای udev برای دستگاه‌ها.
- `settle`: منتظر ماندن برای اتمام پردازش‌های udev.
- `control`: مدیریت وضعیت daemon udev.

## Common Examples
- نمایش اطلاعات یک دستگاه خاص:
```bash
udevadm info --query=all --name=/dev/sda
```

- فعال‌سازی رویدادهای udev برای دستگاه‌ها:
```bash
udevadm trigger
```

- منتظر ماندن برای اتمام پردازش‌های udev:
```bash
udevadm settle
```

- مدیریت وضعیت daemon udev:
```bash
udevadm control --reload-rules
```

## Tips
- همیشه قبل از اعمال تغییرات، از اطلاعات دستگاه‌ها پشتیبان بگیرید.
- برای مشاهده اطلاعات دقیق‌تر، از گزینه‌های مختلف `info` استفاده کنید.
- در صورت بروز مشکلات، می‌توانید از `udevadm monitor` برای نظارت بر رویدادهای udev استفاده کنید.
