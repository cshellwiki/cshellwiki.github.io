# [লিনাক্স] Bash chown ব্যবহার: ফাইলের মালিকানা পরিবর্তন করা

## Overview
`chown` কমান্ডটি লিনাক্স এবং ইউনিক্স সিস্টেমে ফাইল এবং ডিরেক্টরির মালিকানা পরিবর্তন করার জন্য ব্যবহৃত হয়। এটি ব্যবহারকারীদের এবং গ্রুপগুলির মালিকানা পরিবর্তন করতে সক্ষম করে, যা সিস্টেমের নিরাপত্তা এবং অ্যাক্সেস নিয়ন্ত্রণে গুরুত্বপূর্ণ।

## Usage
`chown` কমান্ডের মৌলিক সিনট্যাক্স হল:

```bash
chown [options] [new_owner][:new_group] [file/directory]
```

## Common Options
- `-R`: রিকার্সিভভাবে ফাইল এবং ডিরেক্টরির মালিকানা পরিবর্তন করে।
- `-v`: পরিবর্তনের বিস্তারিত আউটপুট দেখায়।
- `-c`: শুধুমাত্র পরিবর্তিত ফাইলগুলির জন্য আউটপুট দেখায়।

## Common Examples
1. একটি ফাইলের মালিকানা পরিবর্তন করা:
   ```bash
   chown user1 file.txt
   ```

2. একটি ডিরেক্টরির মালিকানা পরিবর্তন করা:
   ```bash
   chown user1 /path/to/directory
   ```

3. মালিকানা এবং গ্রুপ উভয়ই পরিবর্তন করা:
   ```bash
   chown user1:group1 file.txt
   ```

4. একটি ডিরেক্টরির সমস্ত ফাইলের মালিকানা রিকার্সিভভাবে পরিবর্তন করা:
   ```bash
   chown -R user1 /path/to/directory
   ```

5. পরিবর্তনের বিস্তারিত আউটপুট সহ মালিকানা পরিবর্তন করা:
   ```bash
   chown -v user1 file.txt
   ```

## Tips
- নিশ্চিত করুন যে আপনি যথাযথ অনুমতি সহ `chown` কমান্ডটি চালাচ্ছেন, কারণ এটি সাধারণত রুট ব্যবহারকারীর জন্য সীমাবদ্ধ।
- মালিকানা পরিবর্তনের আগে ফাইলের বর্তমান মালিকানা পরীক্ষা করতে `ls -l` কমান্ড ব্যবহার করুন।
- রিকার্সিভ অপশন ব্যবহার করার সময় সতর্ক থাকুন, কারণ এটি সমস্ত সাব-ফাইল এবং ডিরেক্টরির মালিকানা পরিবর্তন করবে।
