# [लिनक्स] Bash shift उपयोग: कमांड लाइन में आर्ग्युमेंट्स को शिफ्ट करना

## Overview
`shift` कमांड का उपयोग कमांड लाइन में दिए गए आर्ग्युमेंट्स को बाईं ओर शिफ्ट करने के लिए किया जाता है। जब आप `shift` कमांड का उपयोग करते हैं, तो यह पहले आर्ग्युमेंट को हटा देता है और बाकी आर्ग्युमेंट्स को एक स्थान बाईं ओर ले जाता है। यह विशेष रूप से स्क्रिप्टिंग में उपयोगी होता है, जहाँ आप आर्ग्युमेंट्स को क्रमबद्ध तरीके से प्रोसेस करना चाहते हैं।

## Usage
`shift` कमांड की मूल सिंटैक्स इस प्रकार है:

```bash
shift [n]
```

यहाँ `n` वैकल्पिक है, और यह बताता है कि कितने स्थानों तक शिफ्ट करना है। यदि `n` नहीं दिया गया है, तो डिफ़ॉल्ट रूप से 1 स्थान शिफ्ट होता है।

## Common Options
- `n`: यह विकल्प यह निर्दिष्ट करता है कि कितने आर्ग्युमेंट्स को शिफ्ट करना है। उदाहरण के लिए, `shift 2` पहले दो आर्ग्युमेंट्स को हटा देगा।

## Common Examples

### उदाहरण 1: बुनियादी शिफ्ट
```bash
#!/bin/bash
echo "पहला आर्ग्युमेंट: $1"
shift
echo "अब पहला आर्ग्युमेंट: $1"
```
इस स्क्रिप्ट में, पहले आर्ग्युमेंट को प्रदर्शित किया जाता है, फिर `shift` का उपयोग करके इसे हटा दिया जाता है।

### उदाहरण 2: कई आर्ग्युमेंट्स को शिफ्ट करना
```bash
#!/bin/bash
echo "पहला आर्ग्युमेंट: $1"
echo "दूसरा आर्ग्युमेंट: $2"
shift 2
echo "अब पहला आर्ग्युमेंट: $1"
```
यहाँ, `shift 2` का उपयोग करके पहले दो आर्ग्युमेंट्स को हटाया गया है।

### उदाहरण 3: लूप के साथ शिफ्ट
```bash
#!/bin/bash
while [ "$#" -gt 0 ]; do
    echo "आर्ग्युमेंट: $1"
    shift
done
```
इस स्क्रिप्ट में, सभी आर्ग्युमेंट्स को एक-एक करके प्रदर्शित किया जाता है जब तक कि कोई आर्ग्युमेंट शेष न रह जाए।

## Tips
- जब आप स्क्रिप्ट में कई आर्ग्युमेंट्स का उपयोग कर रहे हों, तो `shift` का उपयोग करके उन्हें क्रमबद्ध तरीके से प्रोसेस करना आसान होता है।
- ध्यान रखें कि `shift` कमांड के बाद, पहले आर्ग्युमेंट को हटाने के कारण `$1`, `$2`, आदि के मान बदल जाएंगे।
- यदि आप एक स्क्रिप्ट में एक से अधिक बार `shift` का उपयोग कर रहे हैं, तो यह सुनिश्चित करें कि आप सही आर्ग्युमेंट्स को प्रोसेस कर रहे हैं।
