# [Linux] Bash vmstat Использование: мониторинг системных ресурсов

## Обзор
Команда `vmstat` (Virtual Memory Statistics) используется для отображения информации о системной памяти, процессах, вводе/выводе и загрузке системы. Она помогает администраторам и пользователям отслеживать производительность системы и выявлять потенциальные проблемы.

## Использование
Базовый синтаксис команды `vmstat` выглядит следующим образом:

```bash
vmstat [options] [interval] [count]
```

## Общие параметры
- `-a` — отображает информацию о активной и неактивной памяти.
- `-m` — показывает статистику по памяти, выделенной для кеша.
- `-s` — выводит сводную информацию о памяти.
- `-d` — отображает статистику по дискам.
- `interval` — время в секундах между обновлениями.
- `count` — количество обновлений, которые нужно выполнить.

## Общие примеры
1. **Основной вывод статистики:**
   ```bash
   vmstat
   ```

2. **Вывод статистики каждые 5 секунд:**
   ```bash
   vmstat 5
   ```

3. **Вывод статистики 5 раз с интервалом 2 секунды:**
   ```bash
   vmstat 2 5
   ```

4. **Отображение информации о памяти:**
   ```bash
   vmstat -a
   ```

5. **Статистика по дискам:**
   ```bash
   vmstat -d
   ```

## Советы
- Используйте `vmstat` в сочетании с другими командами, такими как `top` или `iostat`, для более полного анализа производительности системы.
- Регулярно мониторьте систему с помощью `vmstat`, чтобы выявлять аномалии в использовании ресурсов.
- Записывайте результаты в файл для дальнейшего анализа, используя перенаправление вывода:
  ```bash
  vmstat 5 > vmstat_output.txt
  ```
