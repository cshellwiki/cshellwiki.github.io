# [Linux] Bash wait उपयोग: प्रक्रिया समाप्त होने की प्रतीक्षा करना

## Overview
`wait` कमांड का उपयोग एक या अधिक पृष्ठभूमि प्रक्रियाओं के समाप्त होने की प्रतीक्षा करने के लिए किया जाता है। यह मुख्य रूप से स्क्रिप्टिंग में उपयोगी होता है, जहां आप यह सुनिश्चित करना चाहते हैं कि एक प्रक्रिया पूरी हो गई है, इससे पहले कि अगली प्रक्रिया शुरू हो।

## Usage
कमांड की मूल संरचना इस प्रकार है:

```bash
wait [options] [arguments]
```

## Common Options
- `-n`: अगली समाप्त होने वाली प्रक्रिया की प्रतीक्षा करें।
- `PID`: विशेष प्रक्रिया आईडी (PID) के लिए प्रतीक्षा करें।

## Common Examples

1. **सभी पृष्ठभूमि प्रक्रियाओं के लिए प्रतीक्षा करना:**
   ```bash
   sleep 5 &
   sleep 3 &
   wait
   echo "सभी प्रक्रियाएँ समाप्त हो गईं।"
   ```

2. **विशेष प्रक्रिया आईडी के लिए प्रतीक्षा करना:**
   ```bash
   sleep 5 &
   pid=$!
   echo "PID: $pid"
   wait $pid
   echo "प्रक्रिया $pid समाप्त हो गई।"
   ```

3. **अगली समाप्त होने वाली प्रक्रिया के लिए प्रतीक्षा करना:**
   ```bash
   sleep 5 &
   sleep 3 &
   wait -n
   echo "अगली प्रक्रिया समाप्त हो गई।"
   ```

## Tips
- जब आप कई पृष्ठभूमि प्रक्रियाएँ चला रहे हों, तो `wait` का उपयोग करके सुनिश्चित करें कि सभी प्रक्रियाएँ समाप्त हो गई हैं।
- `wait` का उपयोग करते समय, यह ध्यान रखें कि यदि आप PID का उपयोग कर रहे हैं, तो यह सुनिश्चित करें कि वह प्रक्रिया अभी भी चल रही है।
- स्क्रिप्ट में `wait` का उपयोग करने से आपके कार्यों के क्रम को नियंत्रित करने में मदद मिलती है, जिससे आप सुनिश्चित कर सकते हैं कि एक कार्य दूसरे कार्य से पहले समाप्त हो गया है।
