# [লিনাক্স] Bash continue ব্যবহার: লুপে পরবর্তী পুনরাবৃত্তিতে চলে যান

## Overview
`continue` কমান্ডটি Bash স্ক্রিপ্টিংয়ে ব্যবহৃত হয় এবং এটি একটি লুপের মধ্যে ব্যবহৃত হয়। যখন এটি কার্যকর হয়, তখন এটি লুপের বর্তমান পুনরাবৃত্তি বন্ধ করে এবং পরবর্তী পুনরাবৃত্তিতে চলে যায়। এটি সাধারণত শর্তাধীন লজিকের সাথে ব্যবহার করা হয়, যেখানে কিছু শর্ত পূরণ হলে লুপের কিছু অংশ এড়িয়ে যেতে হয়।

## Usage
`continue` কমান্ডের মৌলিক সিনট্যাক্স নিম্নরূপ:

```
continue [n]
```

এখানে `n` হল ঐচ্ছিক একটি সংখ্যা যা নির্দেশ করে কতটি স্তর উপরে ফিরে যেতে হবে। যদি `n` উল্লেখ না করা হয়, তবে এটি ডিফল্টভাবে 1 হিসাবে বিবেচিত হয়।

## Common Options
- `n` : এটি ঐচ্ছিক। যদি উল্লেখ করা হয়, তবে এটি নির্দেশ করে কতটি স্তর উপরে ফিরে যেতে হবে। উদাহরণস্বরূপ, `continue 2` নির্দেশ করে যে এটি দ্বিতীয় স্তরের লুপে ফিরে যাবে।

## Common Examples

### উদাহরণ ১: সাধারণ continue ব্যবহার
```bash
for i in {1..5}; do
    if [ $i -eq 3 ]; then
        continue
    fi
    echo $i
done
```
এই উদাহরণে, সংখ্যা ৩ এলে `continue` কমান্ড কার্যকর হবে এবং ৩ ছাড়া বাকী সংখ্যা প্রিন্ট হবে।

### উদাহরণ ২: while লুপে continue ব্যবহার
```bash
count=0
while [ $count -lt 5 ]; do
    count=$((count + 1))
    if [ $count -eq 2 ]; then
        continue
    fi
    echo $count
done
```
এখানে, সংখ্যা ২ এলে `continue` কার্যকর হবে এবং ২ বাদে বাকী সংখ্যা প্রিন্ট হবে।

### উদাহরণ ৩: nested loops এ continue ব্যবহার
```bash
for i in {1..3}; do
    for j in {1..3}; do
        if [ $j -eq 2 ]; then
            continue
        fi
        echo "i=$i, j=$j"
    done
done
```
এই উদাহরণে, `j` এর মান ২ হলে `continue` কার্যকর হবে এবং সেই ক্ষেত্রে `j` এর মান ২ সহ প্রিন্ট হবে না।

## Tips
- `continue` ব্যবহার করার সময় নিশ্চিত হন যে এটি সঠিক লুপে কার্যকর হচ্ছে, কারণ এটি শুধুমাত্র বর্তমান লুপের উপর প্রভাব ফেলে।
- লজিক্যাল শর্তগুলি পরিষ্কারভাবে সংজ্ঞায়িত করুন যাতে `continue` কমান্ডটি সঠিকভাবে কাজ করে।
- লুপের ভিতরে `continue` ব্যবহার করার সময়, এটি কোডের পড়ার যোগ্যতা বাড়াতে সাহায্য করে, তবে অতিরিক্ত জটিলতা এড়িয়ে চলুন।
