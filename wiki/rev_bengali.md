# [লিনাক্স] Bash rev: উল্টো লেখার জন্য কমান্ড

## Overview
`rev` কমান্ডটি একটি টেক্সট ফাইলের প্রতিটি লাইনের অক্ষরগুলোকে উল্টো করে দেয়। এটি সাধারণত টেক্সট প্রক্রিয়াকরণের জন্য ব্যবহৃত হয়, যেখানে আপনি লাইনের অক্ষরগুলোকে বিপরীতভাবে দেখতে চান।

## Usage
`rev` কমান্ডের মৌলিক সিনট্যাক্স হলো:

```bash
rev [options] [arguments]
```

## Common Options
- `-o, --output=FILE` : আউটপুটকে একটি নির্দিষ্ট ফাইলে সংরক্ষণ করে।
- `-h, --help` : সাহায্য তথ্য প্রদর্শন করে।
- `-V, --version` : সংস্করণ তথ্য প্রদর্শন করে।

## Common Examples
নিচে কিছু সাধারণ উদাহরণ দেওয়া হলো:

1. একটি ফাইলের প্রতিটি লাইনের অক্ষর উল্টো করা:
   ```bash
   rev filename.txt
   ```

2. আউটপুটকে একটি নতুন ফাইলে সংরক্ষণ করা:
   ```bash
   rev filename.txt -o reversed.txt
   ```

3. একটি পাইপের মাধ্যমে ইনপুট গ্রহণ করে উল্টো করা:
   ```bash
   echo "Hello World" | rev
   ```

4. একাধিক ফাইলের জন্য উল্টো করা:
   ```bash
   rev file1.txt file2.txt
   ```

## Tips
- `rev` কমান্ডটি টেক্সট ফাইলের প্রতিটি লাইনের অক্ষরগুলোকে দ্রুত উল্টো করতে সহায়ক।
- যদি আপনি একটি বড় ফাইলের উপর কাজ করছেন, তাহলে আউটপুটকে একটি ফাইলে সংরক্ষণ করা ভালো।
- `rev` এর সাথে অন্যান্য কমান্ড যেমন `cat`, `sort` ইত্যাদি ব্যবহার করে আরও জটিল টেক্সট প্রক্রিয়াকরণ করা যেতে পারে।
