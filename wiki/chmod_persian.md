# [لینوکس] Bash chmod استفاده: تغییر مجوزهای فایل

## Overview
دستور `chmod` برای تغییر مجوزهای دسترسی به فایل‌ها و دایرکتوری‌ها در سیستم‌عامل‌های مبتنی بر یونیکس استفاده می‌شود. این دستور به کاربران این امکان را می‌دهد که تعیین کنند چه کسی می‌تواند فایل‌ها را بخواند، بنویسد یا اجرا کند.

## Usage
سینتکس پایه دستور `chmod` به صورت زیر است:

```bash
chmod [options] [arguments]
```

## Common Options
- `-R`: تغییر مجوزها به صورت بازگشتی برای دایرکتوری‌ها و محتویات آن‌ها.
- `-v`: نمایش جزئیات تغییرات مجوزها.
- `-c`: فقط نمایش تغییرات مجوزها در صورتی که تغییراتی صورت گرفته باشد.
- `u`: اشاره به کاربر (مالک فایل).
- `g`: اشاره به گروه.
- `o`: اشاره به دیگران.
- `a`: اشاره به همه (کاربر، گروه و دیگران).
- `+`: اضافه کردن مجوز.
- `-`: حذف مجوز.
- `=`: تعیین مجوزها.

## Common Examples
### 1. تغییر مجوز یک فایل به خواندن و نوشتن برای کاربر و خواندن برای گروه و دیگران
```bash
chmod 644 filename.txt
```

### 2. اضافه کردن مجوز اجرای فایل به کاربر
```bash
chmod u+x script.sh
```

### 3. حذف مجوز نوشتن از گروه
```bash
chmod g-w document.txt
```

### 4. تغییر مجوزها به صورت بازگشتی برای یک دایرکتوری
```bash
chmod -R 755 /path/to/directory
```

### 5. نمایش تغییرات مجوزها
```bash
chmod -v 700 secret.txt
```

## Tips
- همیشه قبل از تغییر مجوزها، از وضعیت فعلی آن‌ها با استفاده از دستور `ls -l` مطلع شوید.
- از گزینه `-R` با احتیاط استفاده کنید، زیرا ممکن است مجوزهای ناخواسته‌ای به فایل‌ها و دایرکتوری‌های فرعی اعمال شود.
- برای امنیت بیشتر، مجوزهای فایل‌ها را به حداقل نیاز کاهش دهید.
