# [لینوکس] Bash update-rc.d استفاده: [مدیریت اسکریپت‌های راه‌اندازی]

## Overview
دستور `update-rc.d` برای مدیریت اسکریپت‌های راه‌اندازی در سیستم‌های مبتنی بر دبیان و اوبونتو استفاده می‌شود. این دستور به شما اجازه می‌دهد تا اسکریپت‌های سرویس را به صورت خودکار در زمان راه‌اندازی سیستم فعال یا غیرفعال کنید.

## Usage
سینتکس پایه دستور به صورت زیر است:

```bash
update-rc.d [options] [arguments]
```

## Common Options
- `defaults`: اسکریپت را با تنظیمات پیش‌فرض اضافه می‌کند.
- `remove`: اسکریپت را از راه‌اندازی سیستم حذف می‌کند.
- `enable`: اسکریپت را برای راه‌اندازی فعال می‌کند.
- `disable`: اسکریپت را برای راه‌اندازی غیرفعال می‌کند.

## Common Examples
### اضافه کردن یک اسکریپت به راه‌اندازی
برای اضافه کردن یک اسکریپت به راه‌اندازی با تنظیمات پیش‌فرض:

```bash
sudo update-rc.d myscript defaults
```

### حذف یک اسکریپت از راه‌اندازی
برای حذف یک اسکریپت از راه‌اندازی:

```bash
sudo update-rc.d myscript remove
```

### فعال کردن یک اسکریپت
برای فعال کردن یک اسکریپت در زمان راه‌اندازی:

```bash
sudo update-rc.d myscript enable
```

### غیرفعال کردن یک اسکریپت
برای غیرفعال کردن یک اسکریپت در زمان راه‌اندازی:

```bash
sudo update-rc.d myscript disable
```

## Tips
- همیشه قبل از تغییرات، از فایل‌های پیکربندی خود نسخه پشتیبان تهیه کنید.
- از گزینه `defaults` برای استفاده از تنظیمات استاندارد و جلوگیری از خطاهای احتمالی استفاده کنید.
- برای بررسی وضعیت اسکریپت‌های راه‌اندازی، می‌توانید به دایرکتوری `/etc/rc*.d/` مراجعه کنید.
