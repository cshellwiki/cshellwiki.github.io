# [لینوکس] C Shell (csh) update-rc.d استفاده: [مدیریت اسکریپت‌های راه‌اندازی]

## Overview
دستور `update-rc.d` برای مدیریت اسکریپت‌های راه‌اندازی در سیستم‌های مبتنی بر دبیان و اوبونتو استفاده می‌شود. این دستور به شما این امکان را می‌دهد که اسکریپت‌های سرویس را به صورت خودکار در زمان راه‌اندازی سیستم فعال یا غیرفعال کنید.

## Usage
سینتکس پایه دستور به صورت زیر است:

```csh
update-rc.d [options] [arguments]
```

## Common Options
- `defaults`: اسکریپت را با تنظیمات پیش‌فرض اضافه می‌کند.
- `remove`: اسکریپت را از راه‌اندازی سیستم حذف می‌کند.
- `enable`: اسکریپت را برای راه‌اندازی فعال می‌کند.
- `disable`: اسکریپت را برای راه‌اندازی غیرفعال می‌کند.

## Common Examples
برای اضافه کردن یک اسکریپت به راه‌اندازی با تنظیمات پیش‌فرض:

```csh
update-rc.d myscript defaults
```

برای حذف یک اسکریپت از راه‌اندازی:

```csh
update-rc.d myscript remove
```

برای فعال کردن یک اسکریپت در زمان راه‌اندازی:

```csh
update-rc.d myscript enable
```

برای غیرفعال کردن یک اسکریپت در زمان راه‌اندازی:

```csh
update-rc.d myscript disable
```

## Tips
- همیشه قبل از اعمال تغییرات، از اسکریپت‌های خود نسخه پشتیبان تهیه کنید.
- از گزینه `defaults` برای اضافه کردن اسکریپت‌ها استفاده کنید تا تنظیمات پیش‌فرض به درستی اعمال شوند.
- برای بررسی وضعیت اسکریپت‌های راه‌اندازی، می‌توانید به دایرکتوری `/etc/rc*.d/` مراجعه کنید.
