# [리눅스] Bash wall 사용법: 사용자에게 메시지 전송

## Overview
`wall` 명령어는 시스템의 모든 로그인된 사용자에게 메시지를 전송하는 데 사용됩니다. 이 명령어는 시스템 관리자나 사용자들이 중요한 정보를 공유할 때 유용합니다.

## Usage
기본 구문은 다음과 같습니다:

```bash
wall [options] [arguments]
```

## Common Options
- `-n`: 메시지를 전송할 때 사용자에게 경고를 하지 않습니다.
- `-s`: 메시지를 간단하게 출력합니다.

## Common Examples
1. **기본 메시지 전송**
   ```bash
   wall "서버 점검이 예정되어 있습니다. 잠시 후에 서비스가 중단됩니다."
   ```

2. **파일에서 메시지 전송**
   ```bash
   wall < /path/to/message.txt
   ```

3. **옵션 사용 예시**
   ```bash
   wall -n "긴급: 시스템 재부팅이 필요합니다."
   ```

## Tips
- `wall` 명령어를 사용할 때는 메시지가 모든 사용자에게 전송되므로, 중요한 정보만 포함시키는 것이 좋습니다.
- 시스템 점검이나 유지보수 작업 전에 사용자에게 미리 알리는 데 유용하게 활용할 수 있습니다.
- 메시지를 전송하기 전에 내용을 미리 확인하고, 필요시 파일을 사용하여 더 긴 메시지를 전달하는 것도 좋은 방법입니다.
