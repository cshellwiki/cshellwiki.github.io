# [Linux] Bash vgextend 使用法: ボリュームグループに物理ボリュームを追加する

## Overview
`vgextend` コマンドは、Linux のボリューム管理システムである LVM (Logical Volume Manager) の一部であり、既存のボリュームグループに新しい物理ボリュームを追加するために使用されます。これにより、ボリュームグループのストレージ容量を拡張することができます。

## Usage
基本的な構文は以下の通りです。

```
vgextend [options] [ボリュームグループ名] [物理ボリューム名...]
```

## Common Options
- `-f` : 強制的にボリュームグループを拡張します。
- `-n` : 新しい物理ボリュームを追加する際に、ボリュームグループの名前を指定します。
- `--test` : 実際には変更を加えず、コマンドが成功するかどうかをテストします。

## Common Examples
以下に、`vgextend` コマンドの一般的な使用例を示します。

### 例 1: 新しい物理ボリュームをボリュームグループに追加する
```
vgextend my_volume_group /dev/sdb1
```
このコマンドは、`/dev/sdb1` を `my_volume_group` に追加します。

### 例 2: 複数の物理ボリュームを追加する
```
vgextend my_volume_group /dev/sdb1 /dev/sdc1
```
このコマンドは、`/dev/sdb1` と `/dev/sdc1` の両方を `my_volume_group` に追加します。

### 例 3: 強制的にボリュームグループを拡張する
```
vgextend -f my_volume_group /dev/sdb1
```
このコマンドは、強制的に `my_volume_group` に `/dev/sdb1` を追加します。

## Tips
- 物理ボリュームを追加する前に、必ずそのボリュームが未使用であることを確認してください。
- `--test` オプションを使用して、実際の変更を加える前にコマンドの結果を確認することをお勧めします。
- ボリュームグループの拡張後は、論理ボリュームを拡張することを忘れないでください。
