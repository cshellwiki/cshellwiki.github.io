# [Linux] C Shell (csh) vgextend の使用法: ボリュームグループに物理ボリュームを追加する

## 概要
`vgextend` コマンドは、既存のボリュームグループに新しい物理ボリュームを追加するために使用されます。これにより、ボリュームグループのストレージ容量を拡張することができます。

## 使用法
基本的な構文は次のとおりです。

```
vgextend [オプション] [ボリュームグループ名] [物理ボリューム名]
```

## 一般的なオプション
- `-f`: 強制的に操作を実行します。
- `--test`: 実際には変更を加えず、操作が成功するかどうかをテストします。
- `-n`: 追加する物理ボリュームの数を指定します。

## 一般的な例
以下に、`vgextend` コマンドのいくつかの実用的な例を示します。

### 例 1: 物理ボリュームをボリュームグループに追加する
```
vgextend myvg /dev/sdb1
```
このコマンドは、`/dev/sdb1` という物理ボリュームを `myvg` というボリュームグループに追加します。

### 例 2: 複数の物理ボリュームを追加する
```
vgextend myvg /dev/sdb1 /dev/sdc1
```
このコマンドは、`/dev/sdb1` と `/dev/sdc1` の2つの物理ボリュームを `myvg` に追加します。

### 例 3: 操作をテストする
```
vgextend --test myvg /dev/sdb1
```
このコマンドは、`/dev/sdb1` を `myvg` に追加する操作が成功するかどうかをテストしますが、実際には変更を加えません。

## ヒント
- 物理ボリュームを追加する前に、必ずそのボリュームが未使用であることを確認してください。
- `--test` オプションを使用して、実際の変更を加える前に操作の結果を確認することをお勧めします。
- ボリュームグループの拡張後は、`lvextend` コマンドを使用して論理ボリュームを拡張することを忘れないでください。
