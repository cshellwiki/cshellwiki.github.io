# [لینوکس] Bash dnf استفاده: مدیریت بسته‌ها در سیستم‌های مبتنی بر RPM

## Overview
دستور `dnf` (Dandified YUM) یک ابزار مدیریت بسته برای توزیع‌های لینوکس مبتنی بر RPM است. این ابزار به کاربران اجازه می‌دهد تا بسته‌های نرم‌افزاری را نصب، حذف و به‌روزرسانی کنند. `dnf` به عنوان جایگزینی برای `yum` طراحی شده و عملکرد بهتری در مدیریت وابستگی‌ها و سرعت دارد.

## Usage
سینتکس پایه دستور `dnf` به صورت زیر است:

```bash
dnf [options] [arguments]
```

## Common Options
- `install`: برای نصب بسته‌های جدید.
- `remove`: برای حذف بسته‌های موجود.
- `update`: برای به‌روزرسانی بسته‌های نصب‌شده.
- `search`: برای جستجوی بسته‌ها.
- `info`: برای نمایش اطلاعات مربوط به بسته‌ها.

## Common Examples
### نصب یک بسته
برای نصب یک بسته، از گزینه `install` استفاده کنید:

```bash
dnf install package_name
```

### حذف یک بسته
برای حذف یک بسته، از گزینه `remove` استفاده کنید:

```bash
dnf remove package_name
```

### به‌روزرسانی بسته‌ها
برای به‌روزرسانی تمام بسته‌های نصب‌شده، از گزینه `update` استفاده کنید:

```bash
dnf update
```

### جستجوی بسته‌ها
برای جستجوی یک بسته خاص، از گزینه `search` استفاده کنید:

```bash
dnf search package_name
```

### نمایش اطلاعات بسته
برای نمایش اطلاعات مربوط به یک بسته، از گزینه `info` استفاده کنید:

```bash
dnf info package_name
```

## Tips
- همیشه قبل از نصب یا به‌روزرسانی بسته‌ها، از گزینه `check-update` برای بررسی به‌روزرسانی‌های موجود استفاده کنید.
- برای جلوگیری از مشکلات وابستگی، از گزینه `--best` هنگام نصب بسته‌ها استفاده کنید.
- می‌توانید با استفاده از گزینه `-y`، تأییدیه‌های لازم را خودکار کنید تا فرآیند نصب یا به‌روزرسانی بدون نیاز به تأیید دستی ادامه یابد.
