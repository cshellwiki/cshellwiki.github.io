# [日本語] Bash mtr の使い方: ネットワーク接続のトラブルシューティング

## Overview
mtr（My Traceroute）は、ネットワーク接続のトラブルシューティングに役立つツールです。pingとtracerouteの機能を組み合わせて、ホストへの接続状況をリアルタイムで監視します。

## Usage
基本的な構文は以下の通りです。

```bash
mtr [options] [arguments]
```

## Common Options
- `-r` : レポートモードで実行し、結果を表示後に終了します。
- `-c <count>` : 指定した回数だけパケットを送信します。
- `-i <interval>` : パケット送信の間隔を指定します（デフォルトは1秒）。
- `-p` : ポート番号を指定して、特定のポートに対してテストを行います。

## Common Examples
以下は、mtrコマンドのいくつかの実用的な例です。

1. 基本的なmtrの実行（デフォルトの設定でホストに接続を確認）:
   ```bash
   mtr example.com
   ```

2. レポートモードで実行し、結果を表示する:
   ```bash
   mtr -r example.com
   ```

3. 10回だけパケットを送信する:
   ```bash
   mtr -c 10 example.com
   ```

4. 特定のポート（例: 80）に対してテストを行う:
   ```bash
   mtr -p 80 example.com
   ```

## Tips
- mtrを使用する際は、管理者権限が必要な場合があります。必要に応じて`sudo`を使用してください。
- 結果をファイルに保存したい場合は、リダイレクトを使用して出力をファイルに書き込むことができます。
- ネットワークの問題を診断する際は、複数のホストに対してmtrを実行し、比較することが有効です。
