# [Linux] Bash unxz उपयोग: फ़ाइलों को अनजिप करना

## Overview
`unxz` कमांड का उपयोग `.xz` फ़ाइलों को अनजिप करने के लिए किया जाता है। यह एक संकुचित फ़ाइल प्रारूप है जो डेटा को कम स्थान में संग्रहित करने के लिए उपयोग होता है। `unxz` कमांड का उपयोग करके आप इन फ़ाइलों को उनके मूल रूप में पुनर्स्थापित कर सकते हैं।

## Usage
इस कमांड का मूल स्वरूप निम्नलिखित है:

```
unxz [options] [arguments]
```

## Common Options
- `-k`, `--keep`: मूल `.xz` फ़ाइल को बनाए रखें।
- `-f`, `--force`: यदि फ़ाइल पहले से मौजूद है, तो उसे ओवरराइट करें।
- `-v`, `--verbose`: प्रक्रिया के दौरान विस्तृत जानकारी दिखाएं।

## Common Examples
यहाँ कुछ सामान्य उदाहरण दिए गए हैं:

1. एक साधारण `.xz` फ़ाइल को अनजिप करना:
   ```bash
   unxz file.xz
   ```

2. मूल फ़ाइल को बनाए रखते हुए अनजिप करना:
   ```bash
   unxz -k file.xz
   ```

3. एक फ़ाइल को ओवरराइट करते हुए अनजिप करना:
   ```bash
   unxz -f file.xz
   ```

4. विस्तृत जानकारी के साथ अनजिप करना:
   ```bash
   unxz -v file.xz
   ```

## Tips
- यदि आप सुनिश्चित नहीं हैं कि आप फ़ाइल को ओवरराइट करना चाहते हैं, तो `-k` विकल्प का उपयोग करें ताकि मूल फ़ाइल सुरक्षित रहे।
- बड़े फ़ाइलों के लिए, अनजिपिंग प्रक्रिया के दौरान `-v` विकल्प का उपयोग करना सहायक हो सकता है, जिससे आप प्रगति को ट्रैक कर सकते हैं।
- यदि आप एक स्क्रिप्ट में `unxz` का उपयोग कर रहे हैं, तो `-f` विकल्प का उपयोग करते समय सावधान रहें, क्योंकि यह मौजूदा फ़ाइलों को बिना किसी चेतावनी के ओवरराइट कर सकता है।
