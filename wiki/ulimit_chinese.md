# [Linux] Bash ulimit 用法: 限制用户进程资源

## 概述
`ulimit` 命令用于控制用户进程可以使用的系统资源的限制。通过设置这些限制，用户可以防止某些进程消耗过多的系统资源，从而影响系统的稳定性和性能。

## 用法
基本语法如下：
```bash
ulimit [选项] [参数]
```

## 常用选项
- `-a`：显示所有当前资源限制。
- `-c`：设置或显示核心文件的大小限制。
- `-d`：设置或显示数据段的大小限制。
- `-f`：设置或显示文件大小的限制。
- `-l`：设置或显示可锁定内存的大小限制。
- `-m`：设置或显示物理内存的大小限制。
- `-n`：设置或显示打开文件的最大数量限制。
- `-s`：设置或显示栈的大小限制。
- `-t`：设置或显示进程的最大运行时间限制。
- `-u`：设置或显示用户可创建的最大进程数限制。

## 常见示例
1. 显示所有资源限制：
   ```bash
   ulimit -a
   ```

2. 设置最大打开文件数为 1024：
   ```bash
   ulimit -n 1024
   ```

3. 设置核心文件大小限制为 0（禁用核心转储）：
   ```bash
   ulimit -c 0
   ```

4. 设置用户最大进程数为 200：
   ```bash
   ulimit -u 200
   ```

5. 显示当前数据段大小限制：
   ```bash
   ulimit -d
   ```

## 提示
- 在修改资源限制时，确保了解系统的需求，以免影响应用程序的正常运行。
- 使用 `ulimit -a` 命令可以快速查看当前的资源限制，帮助你做出合理的调整。
- 资源限制通常在用户的 shell 会话中有效，重新登录后可能会恢复为默认值。可以在用户的配置文件中设置持久化的限制。
