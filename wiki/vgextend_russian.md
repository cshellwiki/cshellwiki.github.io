# [Linux] Bash vgextend <Использование>: Расширение группы томов

## Обзор
Команда `vgextend` используется для расширения группы томов (VG) в системе управления логическими томами (LVM). Она позволяет добавлять новые физические тома (PV) в существующую группу томов, что увеличивает доступное пространство для логических томов.

## Использование
Основной синтаксис команды выглядит следующим образом:

```bash
vgextend [options] [имя_группы_томов] [физические_тома]
```

## Общие опции
- `-f`, `--force`: Принудительно выполнить операцию, даже если это может привести к потере данных.
- `-n`, `--noheadings`: Не выводить заголовки в выводе.
- `-v`, `--verbose`: Выводить подробную информацию о процессе выполнения команды.

## Общие примеры
1. Добавление физического тома в группу томов:
   ```bash
   vgextend my_volume_group /dev/sdb1
   ```

2. Добавление нескольких физических томов:
   ```bash
   vgextend my_volume_group /dev/sdb1 /dev/sdc1
   ```

3. Принудительное добавление физического тома:
   ```bash
   vgextend -f my_volume_group /dev/sdb1
   ```

## Советы
- Перед выполнением `vgextend` убедитесь, что физические тома, которые вы добавляете, не содержат данных, которые могут быть потеряны.
- Используйте опцию `-v` для получения дополнительной информации о процессе, особенно если вы сталкиваетесь с проблемами.
- Регулярно проверяйте состояние вашей группы томов с помощью команды `vgdisplay`, чтобы убедиться, что все физические тома правильно добавлены и доступны.
