# [লিনাক্স] Bash sysctl ব্যবহার: সিস্টেম কনফিগারেশন পরিচালনা

## Overview
`sysctl` কমান্ডটি লিনাক্স এবং ইউনিক্স ভিত্তিক সিস্টেমে কার্নেল প্যারামিটারগুলি পড়া এবং পরিবর্তন করার জন্য ব্যবহৃত হয়। এটি সিস্টেমের কার্যকারিতা এবং নিরাপত্তা উন্নত করতে সাহায্য করে।

## Usage
`sysctl` কমান্ডের মৌলিক সিনট্যাক্স হল:

```bash
sysctl [options] [arguments]
```

## Common Options
- `-a`: সমস্ত কার্নেল প্যারামিটার এবং তাদের মান প্রদর্শন করে।
- `-w`: একটি নির্দিষ্ট প্যারামিটার পরিবর্তন করতে ব্যবহৃত হয়।
- `-n`: প্যারামিটারের মান শুধুমাত্র প্রদর্শন করে, অতিরিক্ত তথ্য ছাড়া।
- `-p`: একটি ফাইল থেকে প্যারামিটারগুলি লোড করতে ব্যবহৃত হয়।

## Common Examples
নিচে কিছু সাধারণ উদাহরণ দেওয়া হল:

1. সমস্ত কার্নেল প্যারামিটার দেখার জন্য:
   ```bash
   sysctl -a
   ```

2. একটি নির্দিষ্ট প্যারামিটার পরিবর্তন করার জন্য, যেমন `vm.swappiness`:
   ```bash
   sysctl -w vm.swappiness=10
   ```

3. একটি প্যারামিটারের মান শুধুমাত্র দেখার জন্য:
   ```bash
   sysctl -n vm.swappiness
   ```

4. একটি কনফিগারেশন ফাইল থেকে প্যারামিটারগুলি লোড করার জন্য:
   ```bash
   sysctl -p /etc/sysctl.conf
   ```

## Tips
- `sysctl` কমান্ড ব্যবহারের আগে সিস্টেমের বর্তমান কনফিগারেশন ব্যাকআপ রাখা একটি ভালো অভ্যাস।
- প্যারামিটার পরিবর্তনের পর সিস্টেমের কার্যকারিতা পরীক্ষা করা উচিত।
- পরিবর্তনগুলি স্থায়ী করতে `/etc/sysctl.conf` ফাইলে প্যারামিটারগুলি যুক্ত করুন।
