# [لينكس] Bash test الاستخدام: [اختبار التعبيرات الشرطية]

## Overview
يستخدم أمر `test` في Bash لاختبار التعبيرات الشرطية. يمكن استخدامه للتحقق من وجود الملفات، أو مقارنة القيم، أو تقييم الشروط المنطقية. يعتبر هذا الأمر أساسياً في كتابة سكربتات Bash التي تحتاج إلى اتخاذ قرارات بناءً على شروط معينة.

## Usage
تكون الصيغة الأساسية للأمر كالتالي:
```bash
test [options] [arguments]
```

## Common Options
- `-e`: يتحقق مما إذا كان الملف موجودًا.
- `-f`: يتحقق مما إذا كان الملف هو ملف عادي.
- `-d`: يتحقق مما إذا كان الملف هو مجلد.
- `-z`: يتحقق مما إذا كانت السلسلة فارغة.
- `-n`: يتحقق مما إذا كانت السلسلة غير فارغة.
- `=`: يقارن بين سلسلتين.
- `-lt`: يتحقق مما إذا كانت القيمة الأولى أقل من الثانية.
- `-gt`: يتحقق مما إذا كانت القيمة الأولى أكبر من الثانية.

## Common Examples
### 1. التحقق من وجود ملف
```bash
if test -e myfile.txt; then
    echo "الملف موجود"
else
    echo "الملف غير موجود"
fi
```

### 2. التحقق مما إذا كان ملفًا عاديًا
```bash
if test -f myfile.txt; then
    echo "هذا ملف عادي"
else
    echo "ليس ملفًا عاديًا"
fi
```

### 3. مقارنة سلسلتين
```bash
string1="hello"
string2="world"

if test "$string1" = "$string2"; then
    echo "السلسلتان متساويتان"
else
    echo "السلسلتان غير متساويتين"
fi
```

### 4. التحقق من عدد العناصر في مصفوفة
```bash
array=(1 2 3)

if test ${#array[@]} -gt 0; then
    echo "المصفوفة تحتوي على عناصر"
else
    echo "المصفوفة فارغة"
fi
```

## Tips
- استخدم الأقواس المربعة `[` و `]` كبديل لأمر `test`، مثل: `[ -e myfile.txt ]`.
- تأكد من وجود مسافات بين الأقواس والخيارات والقيم لتجنب الأخطاء.
- استخدم `&&` و `||` لدمج عدة اختبارات في سطر واحد.
