# [Linux] Bash export Использование: Установка переменных окружения

## Обзор
Команда `export` в Bash используется для установки переменных окружения, которые могут быть доступны для дочерних процессов. Это позволяет передавать данные между различными программами и скриптами, запущенными в одной сессии терминала.

## Использование
Основной синтаксис команды выглядит следующим образом:

```bash
export [options] [arguments]
```

## Общие параметры
- `-p`: Показать все экспортированные переменные.
- `VAR=value`: Установить переменную `VAR` со значением `value` и экспортировать её.

## Общие примеры
1. Экспорт простой переменной:
   ```bash
   export MY_VAR="Hello, World!"
   ```

2. Проверка экспортированных переменных:
   ```bash
   export -p
   ```

3. Экспорт переменной с использованием значения:
   ```bash
   export PATH="$PATH:/usr/local/bin"
   ```

4. Установка переменной окружения для одной команды:
   ```bash
   MY_VAR="Temporary Value" export MY_VAR; echo $MY_VAR
   ```

## Советы
- Используйте `export` для переменных, которые должны быть доступны в дочерних процессах.
- Проверяйте экспортированные переменные с помощью `export -p`, чтобы убедиться, что они установлены правильно.
- Помните, что изменения, сделанные с помощью `export`, действуют только в текущей сессии терминала, если не добавлены в конфигурационные файлы, такие как `.bashrc` или `.bash_profile`.
