# [লিনাক্স] Bash wait ব্যবহার: প্রক্রিয়া শেষ হওয়ার জন্য অপেক্ষা করা

## Overview
`wait` কমান্ডটি একটি Bash শেল কমান্ড যা একটি বা একাধিক ব্যাকগ্রাউন্ড প্রক্রিয়ার সম্পন্ন হওয়ার জন্য অপেক্ষা করে। এটি সাধারণত স্ক্রিপ্টে ব্যবহৃত হয় যেখানে একটি প্রক্রিয়া শেষ হওয়ার পর অন্য একটি প্রক্রিয়া চালানো প্রয়োজন।

## Usage
`wait` কমান্ডের মৌলিক সিনট্যাক্স হলো:

```bash
wait [options] [arguments]
```

## Common Options
- `-n`: পরবর্তী প্রক্রিয়া শেষ হওয়ার জন্য অপেক্ষা করে, যদি একাধিক প্রক্রিয়া চলমান থাকে।
- `-p`: একটি নির্দিষ্ট প্রক্রিয়ার PID এর জন্য অপেক্ষা করে।
- `-f`: ফাংশন হিসেবে অপেক্ষা করে।

## Common Examples

### উদাহরণ 1: সাধারণ অপেক্ষা
একটি ব্যাকগ্রাউন্ড প্রক্রিয়া চালানোর পর `wait` ব্যবহার করা:

```bash
sleep 5 &
wait
echo "প্রক্রিয়া শেষ হয়েছে।"
```

### উদাহরণ 2: একাধিক ব্যাকগ্রাউন্ড প্রক্রিয়ার জন্য অপেক্ষা
একাধিক ব্যাকগ্রাউন্ড প্রক্রিয়া চালিয়ে `wait` ব্যবহার করা:

```bash
sleep 3 &
sleep 5 &
wait
echo "সব প্রক্রিয়া শেষ হয়েছে।"
```

### উদাহরণ 3: নির্দিষ্ট PID এর জন্য অপেক্ষা
একটি নির্দিষ্ট PID এর জন্য অপেক্ষা করা:

```bash
sleep 10 &
pid=$!
wait $pid
echo "PID $pid প্রক্রিয়া শেষ হয়েছে।"
```

## Tips
- `wait` কমান্ডটি ব্যবহার করার সময় নিশ্চিত করুন যে ব্যাকগ্রাউন্ড প্রক্রিয়াগুলি সঠিকভাবে শুরু হয়েছে।
- স্ক্রিপ্টের মধ্যে `wait` ব্যবহার করলে এটি কার্যকরভাবে প্রক্রিয়াগুলির সমন্বয় করতে সাহায্য করে।
- `wait` এর সাথে PID ব্যবহার করলে নির্দিষ্ট প্রক্রিয়া সম্পন্ন হওয়ার সময় নির্ভরযোগ্যভাবে অপেক্ষা করতে পারবেন।
